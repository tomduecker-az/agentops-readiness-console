## What These Examples Prove

These examples show the full governed workflow pattern:

```text
Workflow packet
  → workflow map
  → data sensitivity report
  → risk/control matrix
  → HITL design
  → implementation backlog
  → human approval
  → policy-checked write action
  → audit evidence
```

The payment reconciliation example demonstrates a finance/operations workflow.

The customer onboarding example demonstrates a SaaS/customer-operations workflow with different risk patterns, including onboarding-specific risks around customer-facing commitments, implementation scope, integrations, handoffs, and timelines.

To regenerate outputs:

```bash
PYTHONPATH=services/api python -m scripts.export_example_outputs --workflow-id payment_reconciliation --output-dir examples/payment_reconciliation_outputs
PYTHONPATH=services/api python -m scripts.export_example_outputs --workflow-id customer_onboarding --output-dir examples/customer_onboarding_outputs
```

To validate checked-in outputs:

```bash
PYTHONPATH=services/api python -m scripts.validate_example_outputs
```
